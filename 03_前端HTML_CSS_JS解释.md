# 前端 HTML、CSS、JavaScript 解释

前端由三个文件组成：

```text
frontend/index.html
frontend/styles.css
frontend/app.js
```

## 1. index.html 负责什么

HTML 负责页面结构。

它决定网页上有什么东西。

例如：

- 左侧导航栏。
- 今日提醒区域。
- 顶部统计卡片。
- 推荐任务区域。
- 添加任务表单。
- 任务列表。
- 打卡记录表。
- 打卡日历。
- 数据分析区域。

### 常见 HTML 标签

| 标签 | 作用 |
|---|---|
| div | 普通容器 |
| aside | 侧边栏 |
| main | 主内容区 |
| section | 页面分区 |
| button | 按钮 |
| input | 输入框 |
| select | 下拉选择框 |
| textarea | 多行文本框 |
| table | 表格 |
| script | 引入 JavaScript |

## 2. styles.css 负责什么

CSS 负责页面样式。

它决定网页好不好看。

例如：

- 背景颜色。
- 卡片样式。
- 按钮颜色。
- 页面布局。
- 进度条。
- 日历格子。
- 手机屏幕适配。

### 本项目常用 CSS 技术

| 技术 | 作用 |
|---|---|
| CSS 变量 | 统一管理颜色 |
| display: grid | 网格布局，适合卡片区域 |
| display: flex | 横向排列按钮和标题 |
| border-radius | 圆角 |
| box-shadow | 卡片阴影 |
| media query | 响应式布局，适配小屏幕 |

## 3. app.js 负责什么

JavaScript 负责页面交互和请求后端。

它做这些事：

1. 请求后端任务列表。
2. 请求后端统计数据。
3. 请求后端提醒数据。
4. 渲染任务卡片。
5. 渲染推荐任务。
6. 渲染打卡日历。
7. 提交添加任务表单。
8. 点击打卡时调用后端接口。
9. 点击删除时调用后端接口。
10. 切换总览、任务、打卡、分析页面。

## 4. 前端如何调用后端

前端使用 `fetch`。

例如获取任务：

```js
fetch('/api/tasks')
```

添加任务：

```js
fetch('/api/tasks', {
  method: 'POST',
  body: 表单数据
})
```

后端返回 JSON，前端再更新页面。

## 5. 为什么前端不保存数据

这个版本是前端 + 后端结构。

所以数据由 C++ 后端保存。

前端只负责显示。

这样分工更清楚：

- 前端：页面、按钮、表单、展示。
- 后端：保存、校验、提醒、统计。

## 6. 页面渲染是什么意思

渲染就是把数据显示到网页上。

例如后端返回任务数组，前端把每个任务变成一张任务卡片：

```js
tasks.map(task => `任务卡片 HTML`)
```

然后放进页面：

```js
el.taskList.innerHTML = htmlText;
```

## 7. 事件绑定是什么意思

事件绑定就是告诉浏览器：用户点击按钮时，要执行哪个函数。

例如：

```js
button.addEventListener('click', function () {
  // 点击后执行这里
});
```

本项目中，添加任务、打卡、删除、刷新、切换页面都用了事件绑定。
