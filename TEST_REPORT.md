# 测试报告

## 一、项目名称

学习养成计划 V2（前端 + C++ 后端）

## 二、测试目的

本次测试主要验证系统是否满足课程项目需求，重点检查任务管理、提醒、打卡、数据统计、前后端通信和页面展示是否正常。

## 三、测试范围

本次测试覆盖以下功能：

1. C++ 后端启动。
2. 前端页面访问。
3. 前端调用后端接口。
4. 添加学习任务。
5. 使用系统推荐任务。
6. 完成任务并打卡。
7. 删除任务。
8. 今日提醒和复习提醒。
9. 数据统计与可视化。
10. 打卡记录表和打卡日历。
11. 后端数据文件保存。
12. 异常输入校验。

## 四、测试环境

- 操作系统：Windows
- 编译器：MinGW g++
- 浏览器：Microsoft Edge / Google Chrome
- 后端端口：8090
- 后端可执行文件：`backend/server.exe`
- 前端访问地址：`http://localhost:8090`

## 五、测试方法

本项目采用手工测试为主，结合编译验证和页面功能验证。

1. 编译测试：检查 C++ 后端是否能编译生成可执行文件。
2. 接口测试：通过前端页面操作间接测试后端接口。
3. 功能测试：按照测试用例逐项验证。
4. 界面测试：检查页面布局、按钮、表格、进度条和日历展示效果。
5. 数据测试：关闭页面后重新打开，检查数据是否仍然存在。

## 八、测试结论

本项目主要功能测试通过，能够实现学习任务的添加、管理、打卡、提醒和统计分析。前端页面可以正常调用 C++ 后端接口，后端可以完成数据保存和统计计算。系统整体满足“学习养成计划”选题的功能要求。

## 九、遗留问题

1. 当前后端是单用户本地服务，没有登录功能。
2. 数据保存使用文本文件，不适合大量用户同时使用。
3. 后端 HTTP 解析较简单，适合课程项目演示，不适合作为正式生产服务器。
4. 复习提醒需要在完成任务后的第 1、3、7 天验证，测试周期较长。

## 十、改进建议

1. 后续可以增加用户登录功能。

3. 后续可以增加任务修改功能。
4. 后续可以增加导出学习报告功能。
5. 后续可以加入更完整的自动化接口测试。
