# C++ 后端基础概念解释

你现在只会基础 C++ 和基础数据结构，所以可以先把这个项目理解成下面这样。

## 1. 后端是什么

后端就是用户看不见、但负责处理数据的程序。

在这个项目里，后端由 C++ 编写，主要做这些事：

1. 接收网页发来的请求。
2. 添加学习任务。
3. 删除学习任务。
4. 完成任务并打卡。
5. 保存任务数据到文本文件。
6. 计算提醒信息。
7. 计算任务完成率、每日统计、主题统计。
8. 把结果返回给网页前端。

## 2. 为什么 C++ 后端需要网络库

因为网页和 C++ 程序不是同一个东西。

网页在浏览器里运行，C++ 后端是一个 exe 程序。

它们之间要交流，就需要通过 HTTP 请求。

所以后端用了 Windows 的 WinSock 网络库。

这就是为什么编译时必须加：

```powershell
g++ server.cpp -std=c++17 -finput-charset=UTF-8 -fexec-charset=UTF-8 -o server.exe -lws2_32
```

最后的 `-lws2_32` 就是链接 Windows 网络库。

## 3. struct Task 是什么

`struct Task` 是任务结构体。

结构体可以把多个字段放在一起。

一个学习任务不仅有名字，还包括主题、截止日期、优先级、是否完成等。

所以用结构体表示最合适。

例如：

```cpp
struct Task {
    int id;
    string title;
    string topic;
    string dueDate;
    string dueTime;
    int priority;
    bool needReview;
    bool completed;
};
```

意思是：每个任务都有编号、标题、主题、日期、时间、优先级、是否复习、是否完成。

## 4. vector<Task> 是什么

`vector<Task> tasks;`

意思是：用一个动态数组保存所有任务。

你可以把它理解成一个可以变长的数组。

添加任务时：

```cpp
tasks.push_back(task);
```

删除任务时：

```cpp
tasks.erase(...);
```

遍历任务时：

```cpp
for (const Task& task : tasks) {
    // 处理每一个任务
}
```

## 5. map 是什么

`map` 是映射表，可以按某个关键字统计。

比如按日期统计每天添加了几个任务：

```cpp
map<string, pair<int, int>> daily;
```

这里 `string` 是日期，例如 `2026-07-07`。

`pair<int, int>` 里第一个数字可以表示添加数，第二个数字可以表示完成数。

## 6. 文件读写是什么

项目没有用数据库，而是用文本文件保存任务。

读取文件：

```cpp
ifstream in(DATA_FILE);
```

写入文件：

```cpp
ofstream out(DATA_FILE);
```

任务保存到：

`study_tasks_v2.txt`

## 7. JSON 是什么

JSON 是一种前端很容易理解的数据格式。

后端把 C++ 任务转换成 JSON 字符串，前端收到后就能显示到网页上。

例如：

```json
{
  "id": 1,
  "title": "复习 C++",
  "completed": false
}
```

## 8. HTTP 接口是什么

接口就是前端访问后端的地址。

例如：

- `GET /api/tasks`：获取任务列表。
- `POST /api/tasks`：添加任务。
- `POST /api/tasks/1/complete`：完成编号为 1 的任务。
- `POST /api/tasks/1/delete`：删除编号为 1 的任务。

前端通过这些接口调用 C++ 后端。
