# 07_项目执行流程精讲

这份文档解释项目真正运行时，每一步发生了什么。

## 1. 用户运行后端

用户运行：

```text
backend/server.exe
```

或者运行：

```text
run_backend.bat
```

后端开始执行 `main()` 函数。

### main 函数做的事情

1. 设置控制台编码。
2. 调用 `loadTasks()` 读取历史任务。
3. 调用 `WSAStartup()` 启动 Windows 网络库。
4. 创建 socket。
5. 绑定 8090 端口。
6. 调用 `listen()` 开始监听。
7. 进入 `while (true)` 循环。
8. 等待浏览器请求。

## 2. 用户打开浏览器

用户访问：

```text
http://localhost:8090
```

浏览器会向后端发送请求：

```text
GET /
```

后端在 `handleClient()` 中判断：

如果请求路径是 `/`，就读取：

```text
../frontend/index.html
```

然后把 HTML 返回给浏览器。

## 3. 浏览器继续请求 CSS 和 JS

HTML 中有：

```html
<link rel="stylesheet" href="styles.css" />
<script src="app.js"></script>
```

所以浏览器还会继续请求：

```text
GET /styles.css
GET /app.js
```

后端读取对应文件并返回。

此时页面样式和交互脚本都加载完成。

## 4. 前端开始请求后端数据

`app.js` 最后会执行：

```js
loadAll();
```

`loadAll()` 会同时请求：

```text
GET /api/tasks
GET /api/stats
GET /api/reminders
GET /api/recommendations
```

含义：

| 接口 | 作用 |
|---|---|
| /api/tasks | 获取任务列表 |
| /api/stats | 获取统计数据 |
| /api/reminders | 获取提醒信息 |
| /api/recommendations | 获取推荐任务 |

## 5. 后端返回 JSON

后端把 C++ 数据转换成 JSON。

例如任务列表可能返回：

```json
[
  {
    "id": 1,
    "title": "复习 C++",
    "topic": "C++",
    "completed": false
  }
]
```

前端收到 JSON 后保存到变量：

```js
tasks = taskData;
stats = statData;
recommendations = recommendationData;
```

## 6. 前端渲染页面

前端调用：

```js
renderAll(reminderData);
```

`renderAll()` 会继续调用：

```js
renderMetrics();
renderReminders(reminders);
renderRecommendations();
renderTasks();
renderRecords();
renderCalendar();
renderAnalysis();
```

这些函数分别更新页面不同区域。

## 7. 用户添加任务流程

用户填写表单后点击“添加任务”。

浏览器触发表单提交事件：

```js
el.taskForm.addEventListener("submit", addTaskFromForm);
```

然后执行 `addTaskFromForm()`。

### 前端做什么

1. 阻止浏览器默认提交。
2. 读取输入框内容。
3. 用 POST 请求发送给后端。

请求地址：

```text
POST /api/tasks
```

### 后端做什么

后端进入 `addTaskAction()`。

流程：

1. `parseForm()` 解析表单。
2. `validateTaskInput()` 校验数据。
3. 创建 `Task task;`。
4. 给 task 填入字段。
5. `tasks.push_back(task);` 加入数组。
6. `saveTasks();` 保存到文件。
7. 返回 `{ "ok": true }`。

### 前端收到成功后做什么

1. 清空表单。
2. 显示提示。
3. 重新调用 `loadAll()`。
4. 页面任务列表和统计全部刷新。

## 8. 用户完成打卡流程

用户点击任务卡片的“打卡”。

前端执行：

```js
completeTask(id)
```

请求：

```text
POST /api/tasks/{id}/complete
```

后端执行：

```cpp
completeTaskAction(id)
```

流程：

1. 根据 id 找任务。
2. 找不到就返回失败。
3. 找到就设置 `completed = true`。
4. 设置 `completedDate = todayDate()`。
5. 保存任务文件。
6. 返回成功。

前端刷新后：

- 任务状态变成已完成。
- 打卡记录表出现该任务。
- 打卡日历当天高亮。
- 完成率更新。

## 9. 用户删除任务流程

用户点击删除按钮。

前端先弹出：

```js
confirm("确定删除这个任务吗？")
```

用户确认后请求：

```text
POST /api/tasks/{id}/delete
```

后端执行：

```cpp
deleteTaskAction(id)
```

后端从 `tasks` 中删除该任务，然后保存文件。

## 10. 今日提醒生成流程

前端请求：

```text
GET /api/reminders
```

后端执行：

```cpp
remindersJson()
```

流程：

1. 获取今天日期。
2. 遍历所有任务。
3. 跳过已完成任务。
4. 计算任务截止日期和今天相差几天。
5. 根据优先级和剩余天数生成提醒。
6. 对已完成且需要复习的任务，判断是否是第 1、3、7 天。
7. 返回提醒数组。

## 11. 数据统计生成流程

前端请求：

```text
GET /api/stats
```

后端执行：

```cpp
statsJson()
```

统计过程：

1. 遍历所有任务。
2. 统计总任务数。
3. 统计完成数。
4. 计算完成率。
5. 按日期统计每日添加数和完成数。
6. 按主题统计完成率。
7. 按优先级统计数量。
8. 按完成日期统计打卡次数。
9. 返回 JSON。

## 12. 为什么这个项目叫前端 + 后端

因为前端和后端职责分开。

### 前端负责

1. 显示页面。
2. 接收用户输入。
3. 点击按钮。
4. 向后端发送请求。
5. 把后端返回的数据展示出来。

### 后端负责

1. 保存数据。
2. 校验数据。
3. 添加、删除、打卡。
4. 计算提醒。
5. 计算统计。
6. 返回 JSON。

这就是前后端结合。

## 13. 一句话讲清楚项目流程

用户在网页上操作，JavaScript 把请求发给 C++ 后端，C++ 后端处理任务数据并保存到文件，然后把 JSON 返回给网页，网页再更新显示。
