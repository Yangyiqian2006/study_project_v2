# 05_C++后端语法与代码作用精讲

这份文档专门解释 `backend/server.cpp`。

解释方式是：

1. 先讲语法。
2. 再讲这段代码在项目里的作用。
3. 最后讲为什么要这样写。

## 1. 头文件 include 是什么

代码开头有很多：

```cpp
#include <winsock2.h>
#include <ws2tcpip.h>
#include <windows.h>
#include <algorithm>
#include <ctime>
#include <fstream>
#include <iostream>
#include <map>
#include <sstream>
#include <string>
#include <vector>
```

### 语法解释

`#include` 是预处理指令。

它的作用是在真正编译前，把某个库的声明引入当前文件。

你可以理解为：

> 我现在要用别人已经写好的工具，所以先把工具说明书拿进来。

### 在本项目中的作用

| 头文件 | 作用 |
|---|---|
| winsock2.h | 使用 Windows socket 网络函数 |
| ws2tcpip.h | 使用 TCP/IP 相关网络功能 |
| windows.h | 设置控制台编码，减少中文乱码 |
| algorithm | 使用 replace、remove_if、max 等算法函数 |
| ctime | 处理日期和时间 |
| fstream | 文件读写 |
| iostream | 控制台输出 cout |
| map | 按日期、主题、优先级统计 |
| sstream | 字符串拼接、拆分、转换 |
| string | 使用 C++ 字符串 |
| vector | 使用动态数组保存任务 |

### 为什么要这样写

如果没有这些头文件，编译器就不知道 `vector`、`map`、`ifstream`、`socket`、`send` 这些东西是什么。

所以头文件是“使用功能前的声明”。

## 2. `using namespace std;` 是什么

```cpp
using namespace std;
```

### 语法解释

C++ 标准库里的很多东西都在 `std` 命名空间中。

完整写法本来是：

```cpp
std::string name;
std::vector<int> nums;
std::cout << "hello";
```

写了 `using namespace std;` 后，可以简写成：

```cpp
string name;
vector<int> nums;
cout << "hello";
```

### 在本项目中的作用

项目里大量使用 `string`、`vector`、`map`、`cout`，所以这样写能让代码短一点。

### 为什么适合课程项目

对于课程作业，简化写法更容易阅读。

但大型项目中，有时会避免全局使用 `using namespace std;`，因为可能和其他库名字冲突。

## 3. `struct Task` 任务结构体

```cpp
struct Task {
    int id;
    string title;
    string topic;
    string dueDate;
    string dueTime;
    int priority;
    bool needReview;
    bool completed;
    string createdDate;
    string completedDate;
    int estimatedMinutes;
    string note;
};
```

### 语法解释

`struct` 是结构体。

结构体可以把多个不同类型的数据放到一起。

如果不用结构体，你可能要写很多分散变量：

```cpp
int id;
string title;
string topic;
string dueDate;
```

但是一个任务有很多属性，把它们放到一个结构体里更清晰。

### 字段解释

| 字段 | 类型 | 含义 |
|---|---|---|
| id | int | 任务编号 |
| title | string | 任务名称 |
| topic | string | 学习主题 |
| dueDate | string | 截止日期 |
| dueTime | string | 截止时间 |
| priority | int | 优先级，1 低，2 中，3 高 |
| needReview | bool | 是否需要复习提醒 |
| completed | bool | 是否完成 |
| createdDate | string | 创建日期 |
| completedDate | string | 完成日期 |
| estimatedMinutes | int | 预计学习时长 |
| note | string | 备注 |

### 在项目中的作用

`Task` 是整个项目最核心的数据结构。

添加任务、删除任务、打卡、提醒、统计，都是围绕 `Task` 做的。

### 为什么要这样设计

因为需求中明确说任务有：

- 主题
- 完成时间
- 优先级
- 是否复习提醒
- 打卡记录
- 统计分析

这些都需要字段保存，所以结构体字段就是从需求来的。

## 4. `vector<Task> tasks` 是什么

```cpp
vector<Task> tasks;
```

### 语法解释

`vector` 是 C++ 标准库中的动态数组。

普通数组长度固定，例如：

```cpp
Task tasks[100];
```

最多只能放 100 个任务。

但 `vector` 可以动态增长：

```cpp
vector<Task> tasks;
tasks.push_back(task);
```

### 在项目中的作用

所有任务都放在这个 `tasks` 里。

例如：

- 添加任务：向 `tasks` 加一个 Task。
- 删除任务：从 `tasks` 删除一个 Task。
- 打卡任务：在 `tasks` 里找到对应任务并修改。
- 统计任务：遍历 `tasks` 计算完成率。

### 为什么要这样写

因为任务数量不是固定的。

用户可能添加 1 个任务，也可能添加 100 个任务。

所以用 `vector` 比固定数组更合适。

## 5. `map` 映射表为什么用于统计

后端统计中有代码使用：

```cpp
map<string, pair<int, int>> daily;
map<string, pair<int, int>> topic;
map<int, int> priorityCount;
map<string, int> punchDays;
```

### 语法解释

`map<Key, Value>` 表示一种键值对结构。

例如：

```cpp
map<string, int> count;
count["C++"] = 3;
```

意思是：主题 C++ 有 3 个任务。

### 在项目中的作用

| 变量 | 作用 |
|---|---|
| daily | 按日期统计添加数和完成数 |
| topic | 按主题统计任务数和完成数 |
| priorityCount | 按优先级统计数量 |
| punchDays | 按日期统计打卡次数 |

### 为什么适合统计

因为统计经常是“按某个东西分类”。

例如：

- 按日期分类。
- 按主题分类。
- 按优先级分类。

`map` 正好适合做这种事情。

## 6. `todayDate()` 函数

```cpp
string todayDate() {
    time_t now = time(nullptr);
    tm* t = localtime(&now);
    char buffer[16];
    strftime(buffer, sizeof(buffer), "%Y-%m-%d", t);
    return buffer;
}
```

### 语法解释

这是一个返回 `string` 的函数。

函数名是 `todayDate`。

它没有参数。

返回值是今天日期。

### 每句解释

```cpp
time_t now = time(nullptr);
```

获取当前时间。

```cpp
tm* t = localtime(&now);
```

把时间转换成本地时间格式。

```cpp
char buffer[16];
```

创建一个字符数组，用来保存格式化后的日期。

```cpp
strftime(buffer, sizeof(buffer), "%Y-%m-%d", t);
```

把日期格式化成 `年-月-日`，例如 `2026-07-07`。

```cpp
return buffer;
```

返回日期字符串。

### 在项目中的作用

用于：

1. 创建任务时记录创建日期。
2. 打卡时记录完成日期。
3. 判断任务是否今天截止。
4. 判断任务是否需要复习提醒。

## 7. `cleanField()` 函数

```cpp
string cleanField(string value) {
    replace(value.begin(), value.end(), '|', '/');
    replace(value.begin(), value.end(), '\r', ' ');
    replace(value.begin(), value.end(), '\n', ' ');
    return value;
}
```

### 语法解释

这个函数接收一个字符串，返回清理后的字符串。

`replace` 是算法库中的函数。

格式是：

```cpp
replace(开始位置, 结束位置, 要替换的字符, 替换成什么字符);
```

### 在项目中的作用

任务保存到文本文件时，用 `|` 分隔字段。

例如：

```text
1|复习C++|C++|2026-07-07
```

如果用户输入的任务名称里也有 `|`，文件格式就乱了。

所以要把用户输入中的 `|` 替换成 `/`。

换行符也要替换成空格，避免一条任务被拆成多行。

## 8. `split()` 函数

```cpp
vector<string> split(const string& text, char delimiter)
```

### 语法解释

这个函数有两个参数：

- `text`：要拆分的字符串。
- `delimiter`：分隔符。

返回值是 `vector<string>`，也就是拆出来的一组字符串。

### 在项目中的作用

读取任务文件时，每一行是这样的：

```text
1|复习C++|C++|2026-07-07|20:00
```

需要按 `|` 拆开。

拆完后就能得到任务的每个字段。

## 9. `jsonEscape()` 函数

这个函数是为了保证 JSON 格式正确。

### 为什么需要

如果任务名称是：

```text
复习 "C++"
```

直接拼 JSON 会变成：

```json
{"title":"复习 "C++""}
```

这样格式会坏掉。

所以要把引号变成：

```text
\"
```

这就叫转义。

## 10. `parseForm()` 函数

前端添加任务时，会发送表单数据。

格式类似：

```text
title=复习C++&topic=C++&priority=2
```

`parseForm()` 把它转换成：

```cpp
map<string, string> data;
data["title"] = "复习C++";
data["topic"] = "C++";
data["priority"] = "2";
```

这样后端就可以用 `data["title"]` 取出任务名称。

## 11. `validDate()` 和 `validTime()`

这两个函数属于输入校验。

### 为什么后端要校验

因为前端校验可以被绕过。

如果用户直接请求后端接口，可能传入错误数据。

所以后端必须再次检查。

### 校验内容

日期必须是：

```text
YYYY-MM-DD
```

时间必须是：

```text
HH:MM
```

优先级必须是：

```text
1、2、3
```

## 12. `loadTasks()` 函数

这个函数从文件读取任务。

### 流程

1. 打开 `study_tasks_v2.txt`。
2. 一行一行读取。
3. 每一行用 `split` 拆成字段。
4. 把字段填入 `Task` 结构体。
5. 把任务加入 `tasks` 数组。
6. 更新 `nextId`。

### 意义

让系统重启后还能恢复之前保存的任务。

## 13. `saveTasks()` 函数

这个函数把任务保存到文件。

### 流程

1. 打开数据文件。
2. 遍历所有任务。
3. 把每个任务的字段用 `|` 拼成一行。
4. 写入文件。

### 什么时候调用

- 添加任务后。
- 删除任务后。
- 打卡后。
- 重置演示数据后。

## 14. `taskJson()` 函数

C++ 的结构体不能直接给 JavaScript 用。

所以需要转换成 JSON 字符串。

例如一个 Task 转成：

```json
{
  "id": 1,
  "title": "复习C++",
  "completed": false
}
```

前端收到 JSON 后，就可以显示到页面。

## 15. `remindersJson()` 函数

这个函数生成提醒。

### 核心逻辑

先计算任务截止日期和今天相差多少天。

如果小于 0：逾期。

如果等于 0：今天截止。

如果高优先级并且 7 天内截止：提醒。

如果中优先级并且 3 天内截止：提醒。

如果低优先级并且 1 天内截止：提醒。

完成后的任务，如果设置了复习提醒，就在第 1、3、7 天提醒复习。

## 16. `statsJson()` 函数

这个函数计算统计数据。

### 统计哪些内容

1. 总任务数。
2. 已完成数量。
3. 未完成数量。
4. 完成率。
5. 预计学习总分钟数。
6. 每天添加几个任务。
7. 每天完成几个任务。
8. 每个主题完成率。
9. 每种优先级数量。
10. 每天打卡次数。

### 为什么在后端统计

因为后端负责数据逻辑。

前端只负责展示。

这就是前后端分工。

## 17. `addTaskAction()` 函数

这是添加任务接口。

### 流程

1. 解析前端提交的数据。
2. 调用 `validateTaskInput` 校验。
3. 创建 Task。
4. 给 Task 填入各个字段。
5. 加入 `tasks` 数组。
6. 调用 `saveTasks()` 保存。
7. 返回 JSON 给前端。

## 18. `completeTaskAction()` 函数

这是打卡接口。

### 流程

1. 根据 id 找任务。
2. 找不到就返回失败。
3. 找到就把 `completed` 改成 true。
4. 把 `completedDate` 设置为今天。
5. 保存文件。
6. 返回成功。

## 19. `deleteTaskAction()` 函数

这是删除任务接口。

它使用：

```cpp
remove_if(...)
erase(...)
```

这是 C++ 中常见的 vector 删除写法。

意思是：先把符合条件的元素移动到后面，再真正删除。

## 21. `handleClient()` 函数

这是接口分发中心。

它根据请求路径决定做什么。

例如：

```cpp
GET /api/tasks
```

返回任务列表。

```cpp
POST /api/tasks
```

添加任务。

```cpp
POST /api/tasks/1/complete
```

完成任务。

```cpp
GET /
```

返回前端首页。

## 22. `main()` 函数

主函数流程：

1. 设置中文编码。
2. 加载任务。
3. 启动网络库。
4. 创建服务器 socket。
5. 绑定端口。
6. 开始监听。
7. 无限循环等待请求。
8. 每次收到请求就调用 `handleClient()`。

这就是 C++ 后端能一直运行的原因。
