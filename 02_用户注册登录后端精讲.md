# 02_用户注册登录后端精讲

## 1. User 结构体

```cpp
struct User {
    string username;
    string password;
    string displayName;
    string createdDate;
};
```

### 语法

`struct` 是结构体。

它把多个字段放到一个整体里。

### 作用

User 表示一个系统用户。

字段含义：

| 字段 | 含义 |
|---|---|
| username | 用户名 |
| password | 密码 |
| displayName | 显示名称 |
| createdDate | 注册日期 |

### 意义

有了 User，系统才能判断谁登录了。

## 2. users 动态数组

```cpp
vector<User> users;
```

### 语法

`vector<User>` 是保存 User 的动态数组。

### 作用

系统启动后，会把 `users_v3.txt` 中的用户读入 `users`。

注册新用户时，也会向 `users` 里添加一个 User。

## 3. findUser 函数

```cpp
User* findUser(const string& username)
```

### 语法

返回值是 `User*`，也就是 User 指针。

如果找到用户，返回该用户地址。

如果找不到，返回 `nullptr`。

### 作用

注册时用它判断用户名是否已经存在。

登录时用它查找用户并验证密码。

## 4. loadUsers 函数

### 作用

程序启动时从 `users_v3.txt` 读取用户。

### 流程

1. 打开用户文件。
2. 一行一行读取。
3. 用 `|` 分隔字段。
4. 创建 User。
5. 放入 `users` 数组。

## 5. saveUsers 函数

### 作用

注册新用户后，把 users 数组写入文件。

### 为什么需要

如果不保存文件，程序关闭后用户账号就丢失了。

## 6. authRegister 函数

```cpp
string authRegister(const string& body)
```

### 作用

处理注册请求。

### 流程

1. `parseForm(body)` 解析表单。
2. 取出 username、password、displayName。
3. 判断用户名至少 3 个字符。
4. 判断密码至少 4 个字符。
5. 调用 `findUser(username)` 判断是否重复。
6. 创建 User。
7. `users.push_back(user)` 保存到数组。
8. `saveUsers()` 写入文件。
9. 返回 JSON 给前端。

## 7. authLogin 函数

### 作用

处理登录请求。

### 流程

1. 解析用户名和密码。
2. 查找用户。
3. 如果用户不存在，返回错误。
4. 如果密码不一致，返回错误。
5. 如果正确，返回用户信息。

## 8. 为什么密码明文保存

这是课程项目，为了让你能理解代码，没有引入密码哈希库。
