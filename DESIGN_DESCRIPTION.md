# 项目详细设计说明

## 1. 项目概述

学习养成计划 V2 是一个前端 + C++ 后端结构的课程项目。

系统围绕学习任务管理展开，提供任务添加、任务提醒、复习提醒、完成打卡、打卡日历、进度统计和任务分析功能。

本版本的设计目标不是追求复杂框架，而是在不明显提高学习难度的情况下，让项目结构更接近真实软件开发。

因此，本项目没有使用 Vue、React、Spring、数据库等较复杂技术，而是使用：

- C++ 编写后端服务。
- HTML 编写页面结构。
- CSS 编写页面样式。
- JavaScript 编写前端交互。
- 文本文件保存后端数据。

## 2. 系统架构

系统采用浏览器 + 本地后端服务结构。

用户打开浏览器访问 `http://localhost:8090`。

浏览器加载前端页面。

前端页面通过 JavaScript 调用后端接口。

C++ 后端接收请求，处理任务数据。

后端把结果以 JSON 格式返回给前端。

前端根据返回的数据重新渲染页面。

## 3. 前端模块设计

### 3.1 页面总览模块

页面总览模块展示系统整体学习情况。

它包含四个指标卡：

1. 总任务数。
2. 已完成任务数。
3. 完成率。
4. 预计学习时长。

这些数据不是前端自己计算出来的，而是来自 C++ 后端的 `/api/stats` 接口。

前端只负责把这些数据展示出来。

### 3.2 今日提醒模块

今日提醒模块位于页面左侧。

它展示后端计算出的提醒结果。

提醒规则包括：

1. 逾期任务提醒。
2. 今天截止任务提醒。
3. 高优先级任务提前 7 天提醒。
4. 中优先级任务提前 3 天提醒。
5. 低优先级任务提前 1 天提醒。
6. 复习提醒。

前端通过 `/api/reminders` 获取提醒数组。

如果没有提醒，页面显示“今天没有需要提醒的任务”。

### 3.3 推荐任务模块

推荐任务模块展示后端提供的系统推荐任务。

每个推荐任务包括：

1. 任务名称。
2. 学习主题。
3. 优先级。
4. 预计时长。
5. 备注说明。

用户点击“采用”按钮后，前端会把推荐任务填入添加任务表单。

这一步不会立即保存任务。

用户还可以修改日期、时间和备注后再提交。

### 3.4 任务添加模块

任务添加模块是一个表单。

表单字段包括：

1. 任务名称。
2. 学习主题。
3. 完成日期。
4. 完成时间。
5. 优先级。
6. 预计时长。
7. 备注。
8. 是否需要复习提醒。

用户提交表单后，前端将数据通过 `POST /api/tasks` 发送给后端。

后端负责校验和保存。

如果校验失败，前端显示后端返回的错误信息。

### 3.5 任务列表模块

任务列表模块展示所有任务。

每个任务显示：

1. 任务名称。
2. 学习主题。
3. 优先级。
4. 截止日期和时间。
5. 预计时长。
6. 是否需要复习。
7. 完成状态。
8. 备注。

任务列表支持前端筛选。

筛选条件包括：

1. 全部任务。
2. 未完成任务。
3. 已完成任务。
4. 高优先级任务。

筛选功能只影响页面显示，不修改后端数据。

### 3.6 打卡模块

用户点击任务卡片中的“打卡”按钮后，前端调用：

`POST /api/tasks/{id}/complete`

后端根据任务编号找到对应任务。

如果任务存在，后端将任务状态改为已完成。

同时后端记录完成日期。

完成后前端重新加载数据。

### 3.7 删除模块

用户点击“删除”按钮后，前端先弹出确认框。

用户确认后，前端调用：

`POST /api/tasks/{id}/delete`

后端删除对应任务。

删除成功后前端刷新任务列表和统计数据。

### 3.8 打卡记录模块

打卡记录模块展示所有已完成任务。

显示字段包括：

1. 完成日期。
2. 任务名称。
3. 学习主题。
4. 预计时长。

打卡记录不是单独保存的表，而是从任务列表中筛选已完成任务得到。

### 3.9 打卡日历模块

打卡日历模块按月份展示打卡情况。

后端统计每天的打卡次数。

前端根据 `punchDays` 数据生成日历。

如果某一天有打卡记录，该日期会高亮显示。

如果一天完成多个任务，会显示当天打卡次数。

### 3.10 数据分析模块

数据分析模块包含两个部分：

1. 每日任务分析。
2. 主题完成情况。

每日任务分析展示每天添加的任务数量、完成数量和完成率。

主题完成情况展示每个学习主题下的任务总数、完成数和完成率。

这些统计数据由 C++ 后端完成计算。

前端只负责渲染成卡片和进度条。

## 4. 后端模块设计

### 4.1 HTTP 服务模块

后端使用 WinSock 创建本地 HTTP 服务。

服务监听端口为 8090。

浏览器访问 `http://localhost:8090` 时，后端会返回前端页面。

前端请求 `/api/...` 时，后端返回 JSON 数据。

### 4.2 静态文件模块

后端可以读取前端文件并返回给浏览器。

例如：

1. `/` 返回 `frontend/index.html`。
2. `/styles.css` 返回 `frontend/styles.css`。
3. `/app.js` 返回 `frontend/app.js`。

这样用户不需要单独启动前端服务器。

只需要运行 C++ 后端即可。

### 4.3 任务数据模块

后端使用 `struct Task` 表示任务。

任务字段包括：

1. `id`：任务编号。
2. `title`：任务名称。
3. `topic`：学习主题。
4. `dueDate`：完成日期。
5. `dueTime`：完成时间。
6. `priority`：优先级。
7. `needReview`：是否需要复习。
8. `completed`：是否完成。
9. `createdDate`：创建日期。
10. `completedDate`：完成日期。
11. `estimatedMinutes`：预计时长。
12. `note`：备注。

### 4.4 数据保存模块

后端使用文本文件保存任务。

文件名为：

`study_tasks_v2.txt`

每一行保存一个任务。

字段之间使用 `|` 分隔。

程序启动时读取文件。

任务变化时重新写入文件。

### 4.5 输入校验模块

后端会校验用户输入。

校验内容包括：

1. 任务名称不能为空。
2. 学习主题不能为空。
3. 日期必须合法。
4. 时间必须合法。
5. 优先级必须在 1 到 3 之间。

输入校验放在后端更可靠。

即使前端被绕过，后端也能拦截错误数据。

### 4.6 提醒模块

后端根据任务状态、截止日期和优先级生成提醒。

提醒逻辑包括：

1. 未完成且逾期：显示已逾期。
2. 未完成且今天截止：显示今天截止。
3. 高优先级且 7 天内截止：提示每天推进。
4. 中优先级且 3 天内截止：提示截止时间较近。
5. 低优先级且 1 天内截止：提示临近截止。
6. 已完成且需要复习：在第 1、3、7 天提醒复习。

### 4.7 统计模块

后端统计以下数据：

1. 总任务数。
2. 已完成任务数。
3. 未完成任务数。
4. 总完成率。
5. 预计总学习时长。
6. 每日添加数。
7. 每日完成数。
8. 每日完成率。
9. 按主题统计完成率。
10. 按优先级统计任务数量。
11. 按日期统计打卡次数。

统计结果通过 `/api/stats` 返回。

## 5. 前后端接口设计

### 5.1 获取任务列表

接口：`GET /api/tasks`

作用：获取所有任务。

返回：任务数组。

### 5.2 获取推荐任务

接口：`GET /api/recommendations`

作用：获取系统推荐任务。

返回：推荐任务数组。

### 5.3 获取提醒信息

接口：`GET /api/reminders`

作用：获取今日提醒和复习提醒。

返回：字符串数组。

### 5.4 获取统计信息

接口：`GET /api/stats`

作用：获取任务统计、每日统计、主题统计和打卡统计。

返回：统计对象。

### 5.5 添加任务

接口：`POST /api/tasks`

作用：添加一个新任务。

提交方式：表单数据。

返回：成功或失败信息。

### 5.6 完成任务

接口：`POST /api/tasks/{id}/complete`

作用：把指定任务标记为完成。

返回：成功或失败信息。

### 5.7 删除任务

接口：`POST /api/tasks/{id}/delete`

作用：删除指定任务。

返回：成功或失败信息。

### 5.8 生成演示数据

接口：`POST /api/demo/reset`

作用：生成几条演示任务。

返回：成功信息。

## 7. 项目优点

1. 前后端职责清晰。
2. C++ 不再只是控制台程序，而是真正承担后端数据处理工作。
3. 前端界面比控制台更直观。
4. 技术栈仍然比较基础，适合课程项目。
5. 不依赖复杂框架，方便理解和答辩。
6. 数据可以持久保存。
7. 项目材料包含测试用例、测试报告和缺陷跟踪。

## 8. 项目不足

1. 后端 HTTP 服务比较简单。
2. 当前只支持单用户使用。
3. 文本文件存储不适合大量数据。
4. 没有任务修改功能。
5. 没有登录和权限控制。
6. 没有真正的数据库。
