# 04_任务按用户隔离精讲

## 1. 什么叫用户数据隔离

用户数据隔离就是：

- alice 登录后，只看到 alice 的任务。
- bob 登录后，只看到 bob 的任务。
- alice 不能删除 bob 的任务。
- bob 不能打卡 alice 的任务。

## 2. 后端如何实现隔离

核心是 Task 中新增：

```cpp
string username;
```

每个任务都有所属用户。

## 3. 添加任务时绑定用户

前端提交任务时带上：

```js
username: currentUser.username
```

后端保存任务时：

```cpp
task.username = username;
```

## 4. 查询任务时过滤用户

```cpp
for (const Task& task : tasks) {
    if (task.username == username) result.push_back(task);
}
```

这表示只拿当前用户的任务。

## 5. 统计时过滤用户

统计函数中会判断：

```cpp
if (task.username != username) continue;
```

意思是：如果任务不是当前用户的，就跳过。

## 6. 删除和打卡时也要检查 username

如果只检查 id，可能出现用户 A 操作用户 B 任务的问题。

所以 V3 写成：

```cpp
if (task.id == id && task.username == username)
```

这样更安全。

## 7. 为什么不用多个任务文件

也可以设计成：

```text
alice_tasks.txt
bob_tasks.txt
```

但这样文件会很多，管理更麻烦。

V3 选择所有任务放在一个文件里，用 username 字段区分用户。

这种设计更接近数据库中的“外键关联”。

## 8. 和数据库 ER 图的关系

可以理解为：

- USER 表中的 username 是主键。
- TASK 表中的 username 是外键。
- 一个用户可以拥有多个任务。

关系是：

```text
USER 1 —— N TASK
```
