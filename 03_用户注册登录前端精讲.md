# 03_用户注册登录前端精讲

## 1. 登录注册页面

V3 的 `index.html` 新增了：

```html
<section id="authView" class="auth-page">
```

这个区域就是登录注册页面。

登录成功前，用户只能看到这个区域。

## 2. 登录表单

```html
<form id="loginForm" class="auth-form">
```

用户输入：

- 用户名
- 密码

提交后，JavaScript 调用 `/api/login`。

## 3. 注册表单

```html
<form id="registerForm" class="auth-form hidden">
```

用户输入：

- 用户名
- 显示名称
- 密码

提交后，JavaScript 调用 `/api/register`。

## 4. hidden 类

```css
.hidden { display: none !important; }
```

这个类用于隐藏元素。

登录成功后隐藏登录页，显示主页面。

退出登录后隐藏主页面，显示登录页。

## 5. currentUser 变量

```js
let currentUser = null;
```

它保存当前登录用户。

登录前是 null。

登录成功后变成：

```js
{
  username: "alice",
  displayName: "Alice",
  createdDate: "2026-07-07"
}
```

## 6. localStorage 保存登录状态

```js
localStorage.setItem("study_plan_v3_user", JSON.stringify(user));
```

这句把当前用户保存到浏览器本地。

刷新页面后仍然可以保持登录状态。

退出登录时：

```js
localStorage.removeItem("study_plan_v3_user");
```

## 7. login 函数

```js
async function login(username, password)
```

### 作用

把用户名和密码发给后端。

如果后端返回成功，就进入系统。

如果失败，就显示错误信息。

## 8. register 函数

```js
async function register(username, displayName, password)
```

### 作用

把注册信息发给后端。

注册成功后，直接保存当前用户并进入系统。

## 9. enterApp 函数

登录成功后执行。

它会：

1. 隐藏登录页面。
2. 显示主页面。
3. 显示当前用户名称。
4. 调用 `loadAll()` 加载任务数据。

## 10. logout 函数

退出登录后执行。

它会：

1. 删除 localStorage 中的用户。
2. 清空 currentUser。
3. 隐藏主页面。
4. 显示登录页面。

## 11. 所有任务请求为什么要带 username

因为后端要知道当前操作属于哪个用户。

例如：

```js
GET /api/tasks?username=alice
```

这样后端只返回 alice 的任务。
